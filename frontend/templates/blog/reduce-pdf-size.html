How to Reduce PDF File Size: 7 Proven Methods

March 27, 20246 min read

A PDF that's too large is a practical problem: email attachments get blocked, cloud storage fills up, slow uploads frustrate clients, and mobile users burn through their data plans. There is no single magic solution to making PDFs smaller — but there are seven concrete techniques that work, and choosing the right combination depends on what's in your document. This guide walks through all seven methods with practical advice on when and how to apply each.

Method 1: Reduce Image Resolution (Downsampling)

Images are by far the largest contributor to PDF file size. A typical PDF created from a Word document with several photos can reach 15–30 MB — but those same images at screen resolution would be less than 2 MB.

Downsampling reduces the pixels-per-inch (DPI) of embedded images. For screen viewing, 72–96 DPI is invisible in quality. For printing on a standard laser printer, 150 DPI is fine. Only professional large-format print requires 300+ DPI. If your PDF will mostly be read on screen or printed on a desktop printer, downsampling to 120 DPI typically reduces file size by 60–80% with no perceptible quality loss.

Best for: PDFs with many photos or high-resolution graphics. Potential saving: 50–90%.

Method 2: Re-encode Images with Lossy Compression

PDFs can embed images in various formats, including lossless formats like TIFF or PNG. Converting embedded images to JPEG at 75–85% quality significantly reduces the data stored per image while keeping visual quality high. The difference between JPEG-85 and a lossless PNG is usually indistinguishable unless you zoom in to 200% or more.

Best for: PDFs created from design applications (InDesign, Illustrator) that embed lossless images by default. Potential saving: 30–70%.

Method 3: Remove Metadata

PDFs often contain significant metadata that serves no purpose in the final document: the author's name, the software used to create it, the operating system, revision history, embedded document thumbnails, and custom properties. For a heavily edited document, this metadata can add several hundred kilobytes to several megabytes.

Stripping metadata is also a privacy best practice — it removes information you might not want to share with the recipient. PDFWorks's Strip PDF Metadata tool handles this in one click.

Best for: Any PDF, but especially documents edited multiple times or created in Microsoft Office. Potential saving: 1–15%.

Method 4: Subset Embedded Fonts

When a font is embedded in a PDF, it typically includes the entire font file — all thousands of characters (glyphs) the font supports. But if your document only uses the Latin alphabet plus a few punctuation marks, you only need a tiny fraction of those glyphs. Font subsetting keeps only the characters actually used in the document.

A full font file can be 300 KB to 2 MB. A subsetted version containing only the characters used might be 20–50 KB. For documents using multiple custom fonts, this can add up significantly.

Best for: PDFs using custom or unusual fonts. Potential saving: 5–30% if multiple fonts are embedded.

Method 5: Flatten Interactive Elements

Interactive PDFs — those with form fields, annotations, layers, or other interactive elements — carry additional overhead. If you have a filled-in PDF form that you're now archiving or distributing as a read-only document, flattening it removes the interactive layer and replaces it with static content. This can significantly reduce file size for complex forms.

Best for: Completed PDF forms, documents with many annotations. Potential saving: 10–40% for interactive documents.

Method 6: Optimise the Internal PDF Structure

PDFs accumulate "garbage data" over time. Each time you edit and save a PDF, the old version of modified objects is often retained alongside the new version (this is part of how PDF's incremental update mechanism works). A PDF that's been saved many times can contain significant amounts of superseded data that's no longer referenced by the current document.

Linearising or fully rewriting a PDF with a "Save As" (rather than "Save") in a PDF editor rewrites the file from scratch, discarding all accumulated dead data. Some tools call this "PDF optimisation" or "reduce file size."

Best for: PDFs that have been edited many times. Potential saving: 5–25%.

Method 7: Convert to PDF/A or Reduce Colour Depth

For archival purposes, converting to PDF/A (PDF for Archiving) can sometimes reduce file size by embedding only what's needed for long-term preservation. Alternatively, converting a colour PDF to grayscale (if colour is not essential) can dramatically reduce file size for image-heavy documents, since grayscale images require only one colour channel instead of three.

Best for: Archival documents, black-and-white scans, or documents where colour is not required. Potential saving: 20–60% for colour-heavy documents converted to grayscale.
