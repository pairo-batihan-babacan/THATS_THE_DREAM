PDF Security and Encryption: A Complete Guide

March 30, 20248 min read

PDF's built-in security features are among the most misunderstood aspects of the format. Many people assume a password-protected PDF is completely secure, while others don't realise that a "locked" PDF they created themselves can be unlocked for legitimate use. This guide covers exactly how PDF security works — the types of passwords, encryption standards, permissions systems, and when each approach is appropriate.

Two Types of PDF Passwords

The PDF specification defines two distinct password types that serve fundamentally different purposes:

User Password (Document Open Password)

A user password (also called the "document open password" or "open password") is required just to open the PDF. If you try to open a PDF with a user password set, Adobe Reader and other viewers will immediately display a password prompt before showing any content. Without the correct password, the document content is completely inaccessible.

This type of password is appropriate for confidential documents where even seeing the content is restricted — for example, a PDF containing personal financial information, medical records, or confidential business data.

Permissions Password (Owner Password)

A permissions password (also called the "owner password" or "master password") controls what users can do with the document after opening it. The document can be opened without this password, but certain actions may be restricted, such as:

  • Printing (or printing at high resolution)
  • Copying text or images
  • Modifying the document
  • Adding or modifying annotations or form fields
  • Extracting pages

This is commonly used to distribute PDFs that can be read freely but not easily modified, watermarked documents sent to clients, or digital copies of copyrighted material.

PDF Encryption Standards

The strength of PDF password protection depends on the encryption algorithm used:

40-bit RC4 (PDF 1.1–1.3)

The earliest PDF encryption, introduced in PDF 1.1. Provides only 40-bit key length, which is considered extremely weak by modern standards. Files encrypted with 40-bit RC4 can typically be broken within seconds using modern hardware. Do not use this for any document that needs real security.

128-bit RC4 (PDF 1.4)

A significant improvement over 40-bit, but RC4 itself has known cryptographic weaknesses. While stronger than 40-bit, this encryption method is still considered obsolete for serious security applications. Files from around 2001–2008 often use this standard.

128-bit AES (PDF 1.6)

The first use of AES (Advanced Encryption Standard) in PDF. AES is fundamentally more secure than RC4. 128-bit AES remains adequate for most business use cases today, though 256-bit is now preferred.

256-bit AES (PDF 1.7 / PDF 2.0)

The current standard for PDF security. 256-bit AES is used by banks, governments, and regulated industries for protecting sensitive data. A correctly implemented 256-bit AES-encrypted PDF is computationally infeasible to brute-force with current technology. This is what you should use for any genuinely sensitive document.

Permissions: What Can Be Restricted?

When setting a permissions password, you can restrict the following actions (though the exact options vary between PDF creation tools):

  • Printing — Deny printing entirely, or allow only low-resolution printing.
  • Content copying — Prevent users from selecting and copying text or images to the clipboard.
  • Editing — Prevent modifications to the document content.
  • Annotations — Prevent adding, modifying, or deleting comments and annotations.
  • Form filling — Allow or deny filling in interactive form fields.
  • Accessibility — Allow or deny screen reader access to the content.

Important caveat: Permission restrictions are only enforced by software that respects them. Compliant PDF viewers like Adobe Reader do enforce permissions, but not all tools do. A technically sophisticated user may be able to bypass permission restrictions — they are a deterrent, not an absolute barrier.

How to Protect a PDF with a Password

To add password protection to a PDF, you need a PDF creation or editing tool that supports encryption. Options include:

  • Adobe Acrobat — The most comprehensive PDF tool. Supports all encryption levels and permissions.
  • LibreOffice — Free open-source office suite with PDF export and encryption support.
  • Microsoft Office — When saving/exporting to PDF from Word, Excel, or PowerPoint, you can set a password via the "Options" dialog in the Save As PDF dialogue.
  • macOS Preview — On Mac, Preview can add password protection when exporting PDF via File → Export as PDF.

Metadata and Hidden Information

Even without password protection, PDFs contain hidden metadata that may reveal sensitive information: the original author's name, the software used, the company name from the operating system, editing history, and more. Before sharing a PDF externally, consider using our Strip PDF Metadata tool to remove this information. This is especially important for documents where author anonymity matters — academic blind reviews, journalist source protection, or competitive tender submissions.

Best Practices for PDF Security

  • Use 256-bit AES encryption for any genuinely sensitive document.
  • Use a strong, unique password — not easily guessable words or dates.
  • Strip metadata before sharing PDFs where author identity should not be disclosed.
  • Remember that permissions restrictions are deterrents, not absolute controls.
  • Keep a secure record of passwords for documents you may need to access later.
  • For highly sensitive documents, consider whether PDF is the right format — or whether documents should remain paper-only.
