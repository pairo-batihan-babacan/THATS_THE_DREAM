How to Create Fillable PDF Forms

March 12, 20248 min read

Fillable PDF forms allow recipients to type information directly into the document, check boxes, select from dropdowns, and even add digital signatures — all within a standard PDF viewer. They're used everywhere: job applications, medical intake forms, tax documents, event registrations, surveys, and order forms. This guide explains the types of form fields available, how to create them, and best practices for distributing forms.

What Is a Fillable PDF Form?

A fillable PDF form uses a technology called AcroForms (introduced in PDF 1.2) or the newer XFA (XML Forms Architecture). AcroForms is the widely supported standard — it works in Adobe Reader, most PDF viewers, and browsers. XFA is an Adobe proprietary technology that's being phased out and should generally be avoided for new forms.

From the user's perspective, a fillable form looks like a static PDF but has interactive fields: you click on them and type. The form data is stored within the PDF structure. You can submit the completed form by email, print it, or save it locally.

Types of Form Fields

Text Fields

The most common form field type. Text fields accept typed input: names, addresses, dates, comments, amounts. They can be single-line or multi-line, with optional character limits, number formatting (for currency fields), date validation, or custom validation rules.

Checkboxes

Boolean fields — checked or unchecked. Used for yes/no answers, acknowledgement statements, opt-in/opt-out choices, and feature selection lists. Each checkbox is independent and multiple checkboxes in a group can all be checked simultaneously.

Radio Buttons

Like checkboxes, but grouped so that only one option in a group can be selected at a time. Used for mutually exclusive choices: Yes/No, gender options, payment method, satisfaction rating. Always provide at least two options in a radio button group.

Dropdown Lists (Combo Boxes)

Present a list of predefined options in a compact dropdown. Users select exactly one option. Can optionally allow custom input if the predefined options don't cover every case. Best for standardising input when there are 5+ options — for 2–4 options, radio buttons are usually more user-friendly.

List Boxes

Similar to dropdowns but display multiple options at once, and can allow multi-select. Best for displaying longer lists where users need to see multiple options simultaneously.

Buttons

Clickable buttons that trigger actions: submit the form via email, reset all fields to default values, print the form, or trigger JavaScript actions. "Submit" and "Reset" buttons are the most common uses.

Signature Fields

Placeholder fields where users can apply a digital signature (using their saved digital ID in Acrobat) or a drawn/typed signature. Signed PDFs can carry legally binding digital signatures when using qualified electronic signatures under eIDAS (EU) or comparable frameworks.

Methods for Creating Fillable PDF Forms

Adobe Acrobat Pro (Most Capable)

The gold standard for creating PDF forms. Acrobat Pro's form creation tools allow you to:

  • Automatically detect form areas in an existing PDF and convert them to fields.
  • Manually place any type of field precisely where you want it.
  • Set validation rules, formatting, calculation formulas, and conditional logic.
  • Add JavaScript for complex interactions (automatically calculate totals, show/hide fields based on selection, etc.).
  • Set tab order for keyboard navigation.
  • Prepare the form for digital signatures.

Cost: Around €20/month for Acrobat Pro. Justified for organisations that create forms regularly.

LibreOffice Writer + Export

LibreOffice Writer can create basic fillable forms using its form controls toolbar. Export as PDF with "Export form fields" checked. Supports text fields, checkboxes, and radio buttons. Free and cross-platform, though less polished than Acrobat Pro.

Microsoft Word + Save as PDF

Word's Developer tab includes form controls (content controls) that can be exported to basic interactive PDF forms. The conversion quality is limited — complex Word form controls don't always translate perfectly to PDF fields. Better suited for simple text entry forms than complex interactive documents.

Google Forms + PDF Export

Google Forms creates web-based forms that can be printed or saved as PDFs, but the result is typically a static printed form rather than a true fillable PDF. For simple data collection, Google Forms itself is easier than creating a fillable PDF.

Practical Form Design Tips

  • Label every field clearly. Users should never wonder what to enter in a field. Place labels directly above or to the left of each field.
  • Use field sizes that match expected input. A first name field should be narrower than a mailing address field. This gives users visual cues about how much text is expected.
  • Set a logical tab order. Users navigating by keyboard (including screen reader users) Tab through fields in order. Ensure this order follows the visual reading order (top to bottom, left to right).
  • Validate input where possible. Use number formatting for numeric fields, date validation for date fields, and character limits where appropriate. This prevents data entry errors without frustrating users.
  • Test your form in multiple PDF viewers — Adobe Reader, browser-based viewers (Chrome, Firefox), and on mobile — before distributing.
  • Flatten before archiving. After receiving a completed form, flatten it (convert interactive fields to static content) before archiving to prevent unintended further editing.

Distributing Forms and Collecting Responses

Once created, there are several ways to distribute a fillable PDF form and collect responses:

  • Email attachment — Send the form as an attachment. Respondents complete it, save it, and return it by email. Simple but can result in incomplete forms from users unfamiliar with PDF form filling.
  • Submit button with email action — A Submit button in the PDF can automatically send the form data (or the whole PDF) to a specified email address when clicked. This requires Adobe Reader (not all viewers support this).
  • Download link on a website — Host the PDF and link to it. Visitors download, complete, and return it.
  • Adobe Acrobat Send & Track — Send for signature and track completion status through Adobe's cloud service.
  • DocuSign or similar — For legally binding e-signature workflows, dedicated signature platforms offer better tracking and legal compliance than basic PDF form submission.

Converting Static PDFs to Fillable Forms

If you have a static PDF with form-like appearance (e.g., a scanned paper form) that you want to make fillable, you have two options: use Adobe Acrobat Pro's "Prepare Form" tool (which attempts to auto-detect and create fields from existing form areas), or manually add fields over the static content. The result is a PDF with interactive fields sitting on top of the static background image.
