How to Extract Images from PDF Files

April 2, 20245 min read

PDF files are excellent for sharing documents, but they're not always easy to work with when you need the images inside. Whether you want to reuse a product photo from a supplier's catalogue, extract charts from a research paper, or retrieve graphics from an old design file, extracting images from PDFs is a common need. This guide explains how to do it effectively and what to expect in terms of quality.

Why Extract Images from PDFs?

Common use cases for extracting images from PDFs include:

  • Content reuse — Repurposing images from reports, catalogues, or presentations for use in new documents.
  • Asset recovery — Retrieving graphics when the original source files (Photoshop, Illustrator, etc.) have been lost.
  • Presentation creation — Pulling charts and infographics from research papers to use in PowerPoint slides.
  • Web publishing — Extracting product images from supplier PDFs to use on an e-commerce site.
  • Archive and backup — Extracting and saving images from historical documents for long-term preservation.
  • Design review — Viewing individual images from a PDF in an image editor for quality assessment.

Two Ways to Get Images from a PDF

Method 1: Export Each PDF Page as an Image

This method converts every page of the PDF into an image file (JPG or PNG). The entire page layout — text, images, backgrounds — is rendered as a single flat image. This is useful when you want to capture the full page design, but you won't be able to separate individual image elements from the page content.

PDFWorks's PDF → Images tool uses this approach. It renders each page at a specified resolution and exports it as a separate image file. A 10-page PDF becomes 10 image files.

Method 2: Extract Embedded Images Directly

PDFs can contain images embedded as discrete resources within the document structure. Extracting these gives you each image as a standalone file — exactly as it was when it was placed in the document, without the surrounding page content. This preserves the original image quality.

Quality Considerations

The quality of extracted images depends on the quality of the original images in the PDF:

  • If the PDF was created from high-resolution originals, extracted images will be high resolution.
  • If the PDF was previously compressed (e.g., shared via email and already reduced), extracted images will reflect that compressed quality — you cannot recover resolution that was lost during compression.
  • For page-rendering extraction, resolution is determined by the DPI setting. 150 DPI is suitable for screen use; 300 DPI produces print-quality images.

JPG vs PNG: Which Format Should I Choose?

When extracting pages as images, you'll typically choose between JPG and PNG:

  • JPG — Lossy compression, smaller file sizes. Ideal for photographs and complex images where slight quality loss is acceptable. Best for web use.
  • PNG — Lossless compression, larger file sizes, supports transparency. Ideal for screenshots, diagrams, text-heavy pages, or any content where sharpness is critical.

For most PDF page extractions, PNG produces cleaner results for text and diagrams. Use JPG only when file size is a priority and the content is photographic.

Tips for Best Results

  • Check the source PDF quality first. If the PDF is already compressed or low-resolution, the extracted images will be too. There's no way to add quality that wasn't there to begin with.
  • For scanned PDFs, use the Document → OCR tool if you want text, not images. OCR converts the scanned image to actual searchable text in a Word document.
  • For large PDFs, extracting all pages as images can produce many large files. Consider whether you need all pages or just specific ones — the Split PDF tool can help you isolate specific pages first.
  • Post-processing in an image editor may be needed if you want to crop, resize, or further optimise the extracted images.
