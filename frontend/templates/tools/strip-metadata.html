What Metadata Is Hidden in Your PDF?

Every PDF file contains metadata — structured data stored within the file that describes when it was created, who created it, and how. This information is invisible in the normal view of the document, but can be read by anyone who checks the document properties or uses a metadata analysis tool.

Common metadata stored in PDFs includes:

  • Author name — Your full name, as stored in the operating system user account used to create the document.
  • Creation date and time — Exact timestamp when the document was first created.
  • Modification date and time — When the document was last edited.
  • Creator application — The software used to create the document (e.g., "Microsoft Word for Microsoft 365" or "Adobe Acrobat Pro 2023").
  • Company or organisation name — If your Word installation is configured with a company name, this appears in the document metadata.
  • Document title, subject, and keywords — If set by the document author.
  • Revision history — Number of times the document has been saved.
  • Embedded thumbnail — A small image preview of the document's first page.

Why Remove PDF Metadata?

There are several legitimate reasons to strip metadata before sharing a PDF:

  • Academic blind review — Journals that conduct anonymous peer review require documents with no author identification. Metadata can reveal authorship even if the name is removed from the visible text.
  • Competitive tender submissions — Some procurement processes require anonymous submissions. Metadata can inadvertently identify the submitting organisation.
  • Journalism and source protection — A journalist publishing a document received from a source may need to remove metadata that could identify the source.
  • Personal privacy — Your full name and employer should not necessarily be embedded in every document you share publicly.
  • Professional polish — "Created with Word Trial" in your document properties is not a great look for client-facing materials.

⚠ Critical Warning: Stripping Metadata Does NOT Redact Document Content

This is a dangerous misconception. Removing metadata only removes the file's information header — author name, creation date, software used, and so on. It does absolutely nothing to the actual text and images on the pages.

If you have tried to "redact" sensitive text in your PDF by drawing black rectangles, shapes, or annotation boxes over it, the underlying text is still in the file after metadata stripping. Anyone who receives the file can recover it by:

  • Selecting and copying text through the annotation layer in most PDF viewers
  • Removing the annotation layer entirely in Adobe Acrobat or free tools like pdftk
  • Extracting all text with tools like pdftotext, Python's pypdf, or Acrobat's "Save as Text"
  • Simply opening the file in a text editor and searching for the "hidden" content
🚨 This exact mistake has caused real information leaks. Government agencies — including the U.S. NSA and DOJ — have published documents with classified or sensitive text "hidden" under annotation boxes, only for the content to be trivially recovered and published by journalists. Stripping metadata beforehand did not help.

If you need to redact document content, the correct approach is:

  1. Use a tool that permanently deletes the underlying text and image data — not one that merely draws over it. Adobe Acrobat Pro's built-in "Redact" feature does this correctly.
  2. After redacting, flatten the PDF to bake the redaction marks as permanent static content and remove all annotation layers.
  3. Strip metadata as a final step for additional privacy.
  4. Verify the result: try to select text in the redacted areas — nothing should be selectable.

Stripping metadata is a privacy best practice for normal document sharing — but for documents with sensitive content you have "redacted," it is not a safety measure. Use it as the final polish step after proper redaction and flattening, not as a substitute for either.
